Streaming API Reference

Complete REST API reference for the deprecated local generator's legacy streaming endpoints. The current Fabric live path is stream-events.ipynb writing directly to Eventhouse; see Phase 6.

Base URL

http://localhost:8000/api

Authentication

Currently no authentication required for local development. For production deployments, implement authentication middleware.

Streaming Control Endpoints

Start Streaming

Start legacy real-time event streaming to the configured target.

POST /stream/start

Request Body:

{
  "duration_minutes": null,
  "event_types": ["receipt_created", "inventory_updated"],
  "burst_override": 100,
  "emit_interval_override": 500
}

Parameters: - duration_minutes (integer, optional): Auto-stop after N minutes. Null = run indefinitely - event_types (array, optional): Filter specific event types (must be valid EventType values). Null = all types - burst_override (integer, optional): Override the configured burst size for this session - emit_interval_override (integer, optional): Override the configured emit interval (ms) for this session

Streaming uses the outbox pattern: the background task drains pending streaming_outbox rows in timestamp order, and generates the next day of fact data only when the outbox is empty. See Streaming Workflow.

Response (200 OK):

{
  "success": true,
  "message": "Event streaming started",
  "operation_id": "streaming_a1b2c3d4",
  "started_at": "2024-01-15T10:30:00Z"
}

Error Responses:

  • 409 Conflict: Streaming already active
  • 400 Bad Request: Invalid event types or missing fact data
  • 400 Bad Request: Azure connection string not configured

Example:

curl -X POST http://localhost:8000/api/stream/start \
  -H "Content-Type: application/json" \
  -d '{
    "emit_interval_override": 1000,
    "burst_override": 50,
    "duration_minutes": 10
  }'

Stop Streaming

Stop the currently active streaming session.

POST /stream/stop

Response (200 OK):

{
  "success": true,
  "message": "Event streaming stopped",
  "operation_id": "streaming_a1b2c3d4"
}

Error Responses:

  • 400 Bad Request: No active streaming session
  • 404 Not Found: Streaming session not found

Example:

curl -X POST http://localhost:8000/api/stream/stop

Get Streaming Status

Get current streaming status and basic statistics.

GET /stream/status

Response (200 OK):

{
  "is_streaming": true,
  "status": "running",
  "uptime_seconds": 450.5,
  "events_sent": 15234,
  "events_per_second": 33.8,
  "last_event_time": "2024-01-15T10:37:30Z"
}

Status Values: - running: Actively streaming events - stopped: No active streaming session - error: Streaming encountered errors

Example:

curl http://localhost:8000/api/stream/status

Pause / Resume Streaming

Pause an active streaming session without losing state, then resume it later.

POST /stream/pause
POST /stream/resume
GET /stream/pause-status

Pause status response (200 OK):

{
  "is_paused": false,
  "pause_count": 0,
  "total_pause_duration_seconds": 0.0,
  "currently_paused_duration": 0.0
}

Error Responses: - 400 Bad Request: No active streamer (start streaming first), or not in a pausable/resumable state

Update Streaming Configuration

Update streaming configuration settings.

PUT /stream/config

Request Body:

{
  "emit_interval_ms": 1000,
  "burst": 50,
  "max_batch_size": 128
}

Parameters (all optional): - emit_interval_ms (integer): Time between bursts - burst (integer): Events per burst - azure_connection_string (string): Legacy Event Hub connection string - max_batch_size (integer): Max events per batch - batch_timeout_ms (integer): Batch timeout

Other realtime settings (retry attempts, circuit breaker, etc.) are updated via PUT /api/config with a full configuration object.

Response (200 OK):

{
  "success": true,
  "message": "Streaming configuration updated successfully"
}

Error Responses:

  • 409 Conflict: Cannot update while streaming is active
  • 400 Bad Request: Invalid configuration values

Example:

curl -X PUT http://localhost:8000/api/stream/config \
  -H "Content-Type: application/json" \
  -d '{"emit_interval_ms": 2000, "burst": 25}'

Validate Connection String

Validate the format of a legacy connection string without opening a connection; returns sanitized metadata (never the key itself).

POST /stream/validate-connection

Request Body:

{
  "connection_string": "Endpoint=sb://...;SharedAccessKeyName=...;SharedAccessKey=...;EntityPath=...",
  "strict": true
}

Response (200 OK):

{
  "valid": true,
  "error": null,
  "message": "Connection string is valid",
  "strict_mode": true,
  "metadata": {
    "endpoint": "sb://retail-analytics.servicebus.windows.net/",
    "namespace": "retail-analytics",
    "key_name": "RootManageSharedAccessKey",
    "entity_path": "retail-events",
    "has_key": true,
    "is_fabric_rti": false
  },
  "sanitized": "Endpoint=sb://...;SharedAccessKey=[REDACTED]..."
}

Validate Streaming Configuration

Validate the current (or a proposed) streaming configuration, including the connection string.

POST /config/validate

Request Body (optional): a full configuration object to validate instead of the current one.

Response (200 OK): errors, warnings, and recommendations arrays plus connection metadata.

Drain Outbox

Drain all pending outbox events to the legacy streaming target with pacing.

POST /stream/outbox/drain

Request Body (optional): an emit-interval override in milliseconds as a bare JSON number:

curl -X POST http://localhost:8000/api/stream/outbox/drain \
  -H "Content-Type: application/json" -d '500'

Response (200 OK):

{
  "success": true,
  "message": "Outbox drain started",
  "operation_id": "drain_outbox_a1b2c3d4",
  "started_at": "2024-01-15T10:50:00Z"
}

Note: This runs as a background task. Use /api/tasks/{operation_id}/status to monitor progress.

Event Envelope Format

All events use a standard envelope format for consistency and traceability.

Envelope Schema

{
  "event_type": "receipt_created",
  "payload": {
    "store_id": 42,
    "customer_id": 15678,
    "receipt_id": "RCT_20240115_xyz789",
    "subtotal": 78.50,
    "tax": 6.92,
    "total": 85.42,
    "tender_type": "CREDIT_CARD",
    "item_count": 8
  },
  "trace_id": "TR_20240115_abc123xyz",
  "ingest_timestamp": "2024-01-15T10:40:15.123Z",
  "schema_version": "1.0",
  "source": "retail-datagen",
  "correlation_id": "SESSION_xyz789",
  "partition_key": "store_42"
}

Envelope Fields: - event_type (string): Event type identifier (see Event Types section) - payload (object): Event-specific data - trace_id (string): Unique trace identifier for tracking - ingest_timestamp (string): ISO-8601 timestamp when event was created - schema_version (string): Event schema version (default: "1.0") - source (string): Source system (default: "retail-datagen") - correlation_id (string, optional): Links related events - partition_key (string, optional): Partition key for the legacy streaming target

Rate Limiting

Some endpoints have rate limiting to prevent abuse:

  • /stream/start: 5 requests per 60 seconds
  • /stream/test: 10 requests per 60 seconds
  • /stream/validate-connection: 20 requests per 60 seconds
  • /config/validate: 20 requests per 60 seconds
  • /disruption/create: 20 requests per 60 seconds

Rate limit exceeded response (429):

{
  "detail": "Rate limit exceeded. Try again later."
}

Error Responses

Standard error response format:

{
  "detail": "Error message describing the issue"
}

Common HTTP Status Codes: - 200 OK: Request successful - 400 Bad Request: Invalid request parameters - 404 Not Found: Resource not found - 409 Conflict: Conflicting state (e.g., streaming already active) - 429 Too Many Requests: Rate limit exceeded - 500 Internal Server Error: Server-side error - 501 Not Implemented: Feature not yet implemented

WebSocket Support (Future)

WebSocket support for real-time event streaming is planned for future releases:

ws://localhost:8000/ws/stream

This will enable browser-based real-time event monitoring without polling.

SDK Examples

Python

import requests

# Start streaming
response = requests.post('http://localhost:8000/api/stream/start', json={
    'emit_interval_override': 1000,
    'burst_override': 50,
    'duration_minutes': 10
})
print(response.json())

# Get status
status = requests.get('http://localhost:8000/api/stream/status')
print(status.json())

# Stop streaming
stop = requests.post('http://localhost:8000/api/stream/stop')
print(stop.json())

JavaScript/Node.js

const axios = require('axios');

const baseURL = 'http://localhost:8000/api';

// Start streaming
async function startStreaming() {
  const response = await axios.post(`${baseURL}/stream/start`, {
    emit_interval_override: 1000,
    burst_override: 50,
    duration_minutes: 10
  });
  console.log(response.data);
}

// Get status
async function getStatus() {
  const response = await axios.get(`${baseURL}/stream/status`);
  console.log(response.data);
}

// Stop streaming
async function stopStreaming() {
  const response = await axios.post(`${baseURL}/stream/stop`);
  console.log(response.data);
}
